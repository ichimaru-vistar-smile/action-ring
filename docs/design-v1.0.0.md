# Action Ring 设计方案

版本：`v1.0.0`

## 1. 产品定义

`Action Ring` 是一个面向 `macOS` 的自定义悬浮程序坞。

它的价值不是“替代系统 Dock”，而是提供一个：
- 可随时呼出
- 排序完全可控
- 长期稳定不变化
- 依赖肌肉记忆快速定位的应用面板

目标用户就是你这种对系统 `Dock` 不满意的人：
- 底部区域视觉干扰大
- 图标过多时查找慢
- 固定位置但不够高效
- 不希望把日常效率入口交给付费黑盒软件

## 2. 核心目标

第一版只做一件事：

让用户按下快捷键后，在极短时间内找到并打开自己常用的 App。

具体目标：
- 比系统 `Dock` 更快找到目标应用
- 图标顺序由用户自己定义
- 布局稳定，不被“最近使用”之类机制打乱
- 本地运行，不依赖网络
- 低权限、低复杂度、低维护成本

## 3. 非目标

`v1.0.0` 明确不做：
- 全局搜索
- 多模式切换
- 云同步
- 插件系统
- 账号与授权体系
- AI 功能
- 复杂脚本自动化入口

说明：
后续如果要扩展自动化，也应该建立在稳定的 App 槽位模型之上，而不是在第一版就引入复杂度。

## 4. 交互设计

推荐形态：`横向悬浮 Dock`

原因：
- 比环形布局更直觉
- 一眼扫视成本更低
- 更接近“自定义程序坞”的认知模型
- 更适合固定排序和快速点击

基础交互：
- 用户按下全局快捷键，面板显示
- 面板默认显示在屏幕中央，后续可扩展为鼠标附近
- 点击某个图标时：
  - 应用未启动：启动应用
  - 应用已启动：切换到前台
- `Esc` 关闭面板
- 点击空白处关闭面板

交互原则：
- 布局固定，不能自动重排
- 不按最近使用排序
- 不做复杂动画
- 尽量减少状态变化

## 5. 第一版功能范围

### 必做

- 全局快捷键呼出/关闭面板
- 固定数量的 App 槽位
- 添加 App
- 删除 App
- 拖拽排序
- 点击图标启动或切换 App
- 本地持久化配置

### 建议范围

槽位数控制在 `8-12` 个之间。

原因：
- 太少不够用
- 太多会重新变成“找图标”
- 这个数量更适合肌肉记忆

### 可选但不优先

- 显示 App 名称
- 鼠标悬停高亮
- 开机自启
- 深浅色适配

## 6. 页面与界面

### 6.1 主面板

包含：
- 一排固定顺序的 App 图标
- 当前聚焦态高亮
- 简单关闭逻辑

视觉要求：
- 图标大
- 留白足够
- 背景半透明
- 边框和阴影克制
- 不做花哨动画

### 6.2 设置页

包含：
- 当前 App 列表
- 添加按钮
- 删除按钮
- 拖拽排序
- 快捷键设置

### 6.3 添加 App 面板

包含：
- 本机应用列表
- 应用图标
- 应用名称
- 选择后加入槽位

## 7. 数据模型

第一版数据结构尽量简单。

```json
{
  "version": "v1.0.0",
  "hotkey": {
    "key": "Space",
    "modifiers": ["command", "option"]
  },
  "items": [
    {
      "id": "com.apple.Safari",
      "bundleId": "com.apple.Safari",
      "name": "Safari",
      "path": "/Applications/Safari.app",
      "order": 1
    }
  ]
}
```

说明：
- `bundleId` 用于识别应用
- `path` 用于兜底启动
- `order` 用于保持固定顺序

## 8. 技术方案

建议使用：
- `Swift`
- `SwiftUI`
- `AppKit`

职责拆分：
- `SwiftUI`：设置页与列表配置界面
- `AppKit`：悬浮面板、窗口层级、焦点控制

核心模块：
- `HotkeyManager`：全局快捷键注册
- `DockOverlayController`：显示/隐藏悬浮面板
- `AppCatalogService`：获取本机应用列表
- `AppLaunchService`：启动或激活应用
- `ConfigStore`：本地配置读写

## 9. 权限与安全

第一版尽量避免高权限。

原则：
- 不联网
- 不上传数据
- 不做账号登录
- 不读取用户敏感内容
- 不做键盘模拟和 UI 控制

权限策略：
- 能不用 `Accessibility` 就不用
- 优先使用系统公开 API 实现应用发现、启动、切换

## 11. 结论

这个项目的正确方向不是“做一个功能更杂的 OrbitRing”，而是做一个：

- 更简单
- 更稳定
- 更可控
- 更适合长期肌肉记忆的个人程序坞

第一阶段的成功标准只有一个：

你能比使用系统 `Dock` 更快地找到目标 App。
