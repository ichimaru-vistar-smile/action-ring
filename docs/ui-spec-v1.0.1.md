# Action Ring UI 规格

版本：`v1.0.1`

## 1. 目标

把已有的环形概念图收敛成可开发的 `macOS` 界面规格。

这份文档只回答三个问题：
- 长什么样
- 多大
- 动起来是什么感觉

## 2. 界面定位

主界面是一个 `居中弹出的环形悬浮程序坞`。

它不是：
- 搜索框
- 桌面组件
- 多窗口控制台

它就是一个用于快速打开或切换固定 App 的瞬时面板。

## 3. 布局规格

### 3.1 图标数量

默认使用 `10` 个固定槽位。

原因：
- 比 `8` 个更实用
- 比 `12` 个更容易形成位置记忆
- 与当前参考图结构接近，迁移成本低

### 3.2 画布参考尺寸

以 `1440 x 900` 画布为基准定义。

缩放原则：
- 所有尺寸按最短边比例缩放
- 最终最小整体缩放不低于 `0.85`
- 最大整体缩放不高于 `1.15`

### 3.3 主环尺寸

- 主环外径：`520pt`
- 主环内径：`320pt`
- 主环厚度：`100pt`
- 图标轨道半径：`210pt`
- 中心区直径：`180pt`

说明：
- 当前参考图中心留白过大，这里已经收缩
- 主环只作为弱视觉结构，不应比图标更抢眼

### 3.4 图标与卡片尺寸

- 图标视觉尺寸：`28pt`
- 图标卡片尺寸：`64 x 64pt`
- 卡片圆角：`18pt`
- 图标点击热区：`76 x 76pt`
- 相邻槽位中心角度：`36°`

说明：
- 热区大于视觉卡片，保证好点
- 卡片尺寸统一，不随 App 图标变化

### 3.5 中心区内容

中心区默认显示：
- 当前高亮 App 名称

可选显示：
- 快捷键提示，例如 `Esc 关闭`

不显示：
- 二级说明
- 复杂状态
- 多行文本

中心文案规格：
- 字号：`20pt`
- 字重：`Semibold`
- 对齐：居中
- 最大宽度：`160pt`

## 4. 视觉规格

### 4.1 整体气质

方向：
- 克制
- 冷静
- 原生感
- 轻微未来感，但不赛博

不采用：
- 重霓虹
- 彩虹分区发光
- 强装饰外环
- 游戏 UI 风格

### 4.2 面板材质

- 面板底色：深色半透明
- 背景模糊：`20-24pt`
- 面板透明度：`0.72`
- 边框：`1pt` 低对比描边
- 面板阴影：柔和、范围大、透明度低

建议视觉效果：
- 接近 `macOS` 的磨砂玻璃
- 能从桌面背景中浮出来
- 但不要像发光海报

### 4.3 颜色建议

基础色：
- 面板：深石墨灰
- 文字：偏暖白
- 默认边框：低饱和蓝灰

状态色：
- hover：冷白提亮
- active：冷蓝灰
- launch feedback：短暂白光，不使用彩色爆闪

原则：
- 状态反馈以亮度变化为主
- 不为不同 App 分配一套独立光色

## 5. 层级规格

从下到上共五层：

1. 背景遮罩层
2. 主环弱结构层
3. 图标卡片层
4. hover / active 状态层
5. 中心文字层

层级原则：
- 中心文字不能压过当前高亮图标
- 主环结构只能辅助定位，不能抢焦点
- 所有反馈都以图标卡片层为主

## 6. 状态规格

### 6.1 默认态

- 图标卡片透明度：`0.92`
- 卡片缩放：`1.0`
- 阴影弱
- 描边低对比

### 6.2 Hover 态

- 卡片缩放：`1.06`
- 边框亮度提升约 `18%`
- 阴影透明度略增
- 图标亮度轻微提升
- 中心文字更新为当前 App 名称

### 6.3 Active 态

指鼠标按下但未释放。

- 卡片缩放：`0.98`
- 边框保持高亮
- 阴影略收缩

### 6.4 Open / Switch 反馈

点击成功后给一次极短反馈：
- 卡片亮度快速上扬
- 外扩一层轻微白色柔光
- 持续 `120ms` 左右后恢复

不要做：
- 旋转
- 粒子
- 彩色扩散

## 7. 动画参数

### 7.1 面板出现

- 动画时长：`180ms`
- 效果：透明度 `0 -> 1`，整体缩放 `0.96 -> 1.0`
- 曲线：`easeOut`

### 7.2 面板消失

- 动画时长：`140ms`
- 效果：透明度 `1 -> 0`，整体缩放 `1.0 -> 0.98`
- 曲线：`easeIn`

### 7.3 Hover 切换

- 动画时长：`90ms`
- 效果：卡片放大、描边提亮、阴影增强
- 曲线：`easeOut`

### 7.4 点击反馈

- 动画时长：`120ms`
- 效果：先轻压再恢复
- 曲线：`spring`，但阻尼要高，避免弹跳感过强

## 8. 参考实现建议

### 8.1 位置计算

10 个槽位均匀分布在圆周上。

起始角建议：
- 顶部中心为 `-90°`

后续按顺时针排列。

### 8.2 容错

当窗口较小或屏幕较小时：
- 自动缩放整体环尺寸
- 保持图标卡片最小尺寸不低于 `56pt`
- 保持中心区最小直径不低于 `148pt`

### 8.3 图标素材

- 使用系统或应用原始图标
- 不重新设计图标风格
- 不加底板外的额外装饰

## 9. 当前结论

最终收敛方向是：

一个 `macOS` 风格、居中弹出的、固定排序的环形悬浮程序坞。

保留参考图的结构优势：
- 环形记忆点强
- 图标位置稳定
- 聚焦明确

去掉参考图的问题：
- 装饰过多
- 发光过重
- 中心区浪费
- 产品真实感不足

## 10. 下一步

基于这份规格进入 `v1.1.0` 原型开发时，只需要实现四件事：
- 悬浮面板
- 环形布局
- Hover / Click 状态
- App 启动与切换
